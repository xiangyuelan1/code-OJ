# OJ 在线评测系统 - 产品需求文档

## 1. 系统概述

### 1.1 项目背景
开发一个功能完善的在线评测系统（OJ系统），支持管理员和普通学生两种角色，系统集成AI功能以提升用户体验和学习效果。

### 1.2 核心价值
- 为编程学习者提供专业的在线评测环境
- 为教育管理者提供高效的题目管理工具
- 通过AI技术实现智能化辅导和反馈

### 1.3 目标用户
- **管理员**：教育机构教师、课程管理者，负责题目管理、题解维护和AI功能配置
- **学生**：编程学习者，通过系统练习编程、提交答案、查看题解

---

## 2. 用户角色与权限

### 2.1 管理员角色
| 权限项 | 描述 |
|--------|------|
| 用户管理 | 查看所有学生用户、禁用/启用账户 |
| 题目管理 | 创建、编辑、删除编程题、选择题、填空题 |
| 题解管理 | 上传、编辑、删除题解 |
| AI配置 | 启用/禁用AI功能，配置API信息 |
| 系统设置 | 查看系统使用统计 |

### 2.2 学生角色
| 权限项 | 描述 |
|--------|------|
| 浏览题目 | 查看题目列表、题目详情 |
| 提交答案 | 提交编程题代码、选择题答案、填空题答案 |
| 查看结果 | 查看判题结果、错误信息 |
| 查看题解 | 查看题解内容（需登录） |
| AI辅导 | 使用AI辅导功能（需管理员启用） |
| 个人中心 | 查看做题记录、提交历史 |

---

## 3. 功能模块详细需求

### 3.1 题目管理模块

#### 3.1.1 题目类型
1. **编程题**
   - 题目描述（支持富文本/Markdown）
   - 输入输出示例
   - 测试用例（多个测试点）
   - 时间限制、内存限制
   - 语言限制（可选）

2. **选择题**
   - 题目描述
   - 选项列表（A/B/C/D）
   - 正确答案
   - 解析（可选）

3. **填空题**
   - 题目描述
   - 答案关键词
   - 部分得分逻辑（可选）

#### 3.1.2 题目属性
- 难度等级：简单、中等、困难
- 标签/分类：数据结构、算法、语言基础等
- 关联知识点

### 3.2 判题系统模块

#### 3.2.1 编程题判题
- 代码编译/解释执行
- 多测试点逐个判定
- 结果比对（完全匹配、Special Judge）
- 性能判定（超时、内存超限）
- 返回判题结果和错误信息

#### 3.2.2 选择题判题
- 答案即时判定
- 返回正确/错误状态

#### 3.2.3 填空题判题
- 关键词匹配判定
- 支持部分得分

### 3.3 题解模块

#### 3.3.1 题解内容
- 详细解题思路
- 代码实现
- 复杂度分析
- 扩展知识

### 3.4 AI功能模块

#### 3.4.1 管理员配置
- AI功能总开关
- API服务商选择（OpenAI、Claude、本地模型等）
- API密钥配置
- API基础地址配置
- 模型选择

#### 3.4.2 AI功能（学生端）
| 功能 | 描述 |
|------|------|
| AI代码解释 | 对学生提交的代码进行逐行解释 |
| AI思路引导 | 提供解题思路提示（不直接给出答案） |
| AI错误诊断 | 分析代码错误原因 |
| AI相似题目推荐 | 根据当前题目推荐类似题目 |

#### 3.4.3 AI功能（管理员端）
| 功能 | 描述 |
|------|------|
| AI题解生成 | 根据题目自动生成题解草稿 |
| AI题目推荐 | 推荐适合添加的新题目类型 |

### 3.5 用户管理模块

#### 3.5.1 认证系统
- 用户注册（学生）
- 用户登录（支持邮箱/用户名）
- 密码重置
- 角色区分（管理员/学生）

#### 3.5.2 管理员账户
- 由系统初始化创建
- 支持多管理员

## 4. 页面结构

### 4.1 公共页面
- 首页（题目列表、搜索、筛选）
- 题目详情页
- 登录/注册页
- 个人中心

### 4.2 学生页面
-做题页面（包含代码编辑器、答案提交）
- 提交历史页
- 判题结果页
- 题解查看页
- AI助手页

### 4.3 管理员页面
- 管理后台首页
- 题目管理（列表、创建、编辑）
- 题解管理
- 用户管理
- AI配置页
- 统计面板

## 5. 技术栈建议

### 5.1 前端
- React 18 + TypeScript
- Vite 构建工具
- TailwindCSS 样式
- Monaco Editor（代码编辑器）
- React Router（路由）

### 5.2 后端
- Node.js + Express/Koa
- TypeScript
- SQLite/PostgreSQL 数据库
- JWT 认证
- 代码沙箱执行环境

### 5.3 AI集成
- OpenAI API
- Claude API（可选）
- 支持自定义API端点

---

## 6. 非功能性需求

### 6.1 性能需求
- 页面加载时间 < 2秒
- 判题响应时间 < 30秒（简单题目）
- 支持100+并发用户

### 6.2 安全需求
- 用户密码加密存储
- JWT Token认证
- 代码沙箱隔离执行
- SQL注入防护
- XSS防护

### 6.3 可用性需求
- 响应式设计（支持PC/平板）
- 友好的错误提示
- 加载状态反馈

---

## 7. 项目优先级

### 第一阶段（核心功能）
1. 用户认证系统
2. 题目管理（三种题型）
3. 基础判题功能
4. 题解功能

### 第二阶段（AI增强）
1. AI配置界面
2. AI代码解释
3. AI思路引导

### 第三阶段（优化完善）
1. 高级判题（Special Judge）
2. 做题统计与分析
3. 用户行为追踪

---

## 8. 验收标准

- [ ] 管理员可成功创建三种类型的题目
- [ ] 学生可提交代码并获得判题结果
- [ ] 管理员可配置AI功能
- [ ] 学生可使用AI辅导功能
- [ ] 系统界面美观、操作流畅
- [ ] 响应式设计，支持不同设备
